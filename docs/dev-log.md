## 過去の主な開発経緯（git log より再構成）

### アプリの骨格（2026年初期）

- 基本的な練習記録機能（開始・終了・保存）を実装
- Google Apps Script + Google Sheets でバックエンドを構築
- GAS URL は `index.html` の `GAS_URL` 定数に記載

### PWA 化

- `manifest.webmanifest`・`sw.js`・アイコンを追加してPWA化
- Service Worker のキャッシュ戦略を「index.html はネットワーク優先、画像・マニフェストはキャッシュ優先」に設計
  - **理由**: 全キャッシュ優先にすると、更新後もスマートフォンが古い index.html を使い続けるバグが発生したため

### 曲管理機能の追加

- 曲をスプレッドシートで管理し、アプリの「曲管理」タブから追加・削除・並び替えができるように
- 曲名とアーティスト名を別列（B列・C列）に分離
- CDN URL（SortableJS）は Service Worker のキャッシュリストに含めると SW インストールが失敗するため除外

### カスタムドロップダウン実装

- Android の `<select>` は CSS でスタイルできないため HTML/CSS/JS でカスタムドロップダウンを独自実装
- `<select>` 廃止後、旧 `select.options` への参照が残っていたバグを修正

### 通算回数・最終練習日の表示

- プルダウンに「X日前・Y回」を表示
- 通算回数は GAS が MAX+1 方式で書き込む（COUNTIF ではなく）
  - **理由**: COUNTIF は過去データインポート時に正しく累積されない

### GAS V8 の instanceof Date バグ修正

- `getValues()` で取得した日付が `instanceof Date` で false になる GAS V8 の挙動を確認
- `typeof val.getTime === 'function'` による duck typing で判定するよう修正
- **症状**: 最終練習日が古い日付のまま誤表示された（文字列化後の曜日の頭文字でアルファベット順比較になっていた）

### 保存確認モーダルの追加

- 「終了・保存」タップ → 確認モーダル →「保存する」でスプレッドシートに送信
- タイマーはモーダル表示中も継続（`endPractice()` では clearInterval しない、`confirmSave()` 内でのみ止める）
- `pendingSaveEndTime` に終了時刻を保持しておき、「練習に戻る」時は null に戻す

### メモ欄の追加

- 保存確認モーダルにメモ欄（`#memo-input`）を追加
- スプレッドシートの I 列（9列目）に保存
- 「練習に戻る」を押してもメモの内容は保持

### 「今回の練習」サマリー表示

- 練習タブ下部に「X曲 ／ 合計 Y分Z秒」を表示
- GAS アクセス不要。`sessionSongs`・`sessionSeconds` のグローバル変数でカウント
- ページリロードでリセット。1曲も保存していない間は非表示
